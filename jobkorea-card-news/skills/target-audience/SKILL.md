# 직무별 타겟 오디언스 정의

## 공통 특성
- 위치: 대한민국 재직자/구직자
- 연차: 신입(0년)~15년차 이상 (마케터는 저연차 중심)
- 공통 니즈: 이직 성공, 커리어 레벨업, 실무 역량 강화

---

## 개발자 방

**직무 구성**
- 주: 백엔드·프론트엔드·웹·앱 개발자
- 소수: 데이터 분석가, 엔지니어

**관심 주제 (높은 순)**
1. 기술 스택 트렌드 (AI 도구, 새 프레임워크)
2. 연봉·이직 타이밍
3. 사이드 프로젝트·포트폴리오
4. 코드 품질·아키텍처
5. 기술 면접 준비

**기피 주제**
- 너무 추상적인 리더십 이야기
- 비개발 직무 관점의 조언
- 오래된 기술 기준의 내용

**언어 스타일**
- 기술 용어 사용 OK (React, CI/CD, MSA 등)
- 직접적·간결한 문체 선호
- 코드·수치 기반 근거 신뢰

**페르소나 설정**
- 저연차(A): 2년차 스타트업 백엔드 개발자, 첫 이직 고민 중
- 고연차(B): 8년차 대기업 프론트엔드 개발자, 테크리드 전환 고민
- HR(C): IT 기업 개발자 채용 담당, 연 50명 이상 채용 경험

---

## PMPO 방

**직무 구성**
- Product Manager, Product Owner
- 사업 기획자, 서비스 기획자

**관심 주제 (높은 순)**
1. PM/PO 커리어 경로·레벨업
2. 프로덕트 프레임워크 (OKR, Jobs-to-be-done 등)
3. 데이터 기반 의사결정
4. 개발자·디자이너 협업
5. 이직 시 포트폴리오 구성

**기피 주제**
- 개발 기술 중심 내용
- 단순 업무 도구 소개 (맥락 없는)
- 스타트업 한정 사례

**언어 스타일**
- 프레임워크·방법론 용어 OK (스프린트, PRD, 유저 스토리)
- 사례 중심 설득 선호
- "왜"에 대한 근거 중시

**페르소나 설정**
- 저연차(A): 2년차 스타트업 PM, PO로 이직 준비 중
- 고연차(B): 9년차 대기업 서비스 기획자, 프로덕트 매니저 전환 고민
- HR(C): 테크 스타트업 PM 채용 담당, 포트폴리오 심사 다수 경험

---

## 디자이너 방

**직무 구성**
- UI/UX 디자이너, 프로덕트 디자이너
- 웹·앱 디자이너, 그래픽 디자이너

**관심 주제 (높은 순)**
1. 포트폴리오 구성·케이스 스터디
2. AI 디자인 툴 활용 (Figma AI, Midjourney 등)
3. UX 리서치 방법론
4. 개발자 핸드오프
5. 디자인 시스템 구축

**기피 주제**
- 너무 마케팅·광고 중심 내용
- 그래픽 아트 중심 (UX와 분리)
- 기술 스택 깊이 들어가는 내용

**언어 스타일**
- 비주얼·감각적 표현 OK
- 사용자 중심 언어 (사용자 경험, 터치포인트)
- 실제 작업물 사례 중시

**페르소나 설정**
- 저연차(A): 2년차 앱 UI/UX 디자이너, 포트폴리오 리뉴얼 고민
- 고연차(B): 8년차 프로덕트 디자이너, 디자인 리드 전환 준비
- HR(C): IT 기업 디자이너 채용 담당, 포트폴리오 심사 기준 보유

---

## 마케터 방

**직무 구성**
- 디지털 마케터 (퍼포먼스·콘텐츠·브랜드)
- 저연차 중심 (0~5년차 다수)

**관심 주제 (높은 순)**
1. 퍼포먼스 마케팅 실무 (Meta·Google 광고)
2. 콘텐츠 마케팅·SNS 운영
3. GA4·데이터 분석 기초
4. AI 마케팅 툴 활용
5. 마케터 커리어 경로

**기피 주제**
- 너무 고연차 관점 (임원·전략 중심)
- 오프라인 마케팅 중심
- B2B 영업 혼합 내용

**언어 스타일**
- 트렌디하고 캐주얼한 문체 OK
- 수치·KPI 기반 설득
- "바로 써먹는" 실용 팁 선호

**페르소나 설정**
- 저연차(A): 1년차 스타트업 디지털 마케터, 퍼포먼스 마케팅 공부 중
- 고연차(B): 5년차 이커머스 퍼포먼스 마케터, 그로스 마케터 전환 고민
- HR(C): 이커머스 마케터 채용 담당, 포트폴리오·KPI 중심 심사
